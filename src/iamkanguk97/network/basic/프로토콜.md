# 프로토콜(Protocol)

> [!TIP]
>
> - 네트워크 내의 호스트는 서로가 주고받을 내용인 패킷을 이해할 수 있어야 하고, 이를 올바르게 주고받을 수 있어야 한다.
> - 이를 위한 개념이 바로 프로토콜이다.

## 개념

```
패킷을 주고받는 두 호스트가 서로 다른 LAN에 속해있어서 서로가 주고받는 패킷이 네트워크 장비를 거친다고 가정해보자.
두 호스트가 주고받는 패킷 속의 내용은 서로가 이해할 수 있어야 한다.
(우리가 대화할 때도 마찬가지로 둘 다 한국말을 사용할 줄 알아야 소통이 편리한 것처럼)

물론, 호스트 뿐만 아니라 패킷이 거쳐갈 네트워크 장비도 패킷의 내용을 이해할 수 있어야 한다.
이러한 상황을 위해서 정해진 규칙이 프로토콜이다.
```

- 프로토콜(Protocol)은 네트워크에서 통신을 주고받는 노드 간의 합의된 규칙이나 방법을 의미한다.
- 패킷을 주고받는 호스트와 네트워크 장비들이 서로 주고받는 정보를 이해하려면 같은 프로토콜을 이해해야 하고, 같은 프로토콜로 통신을 해야한다.

### 예시

> [!TIP]
>
> - 프로토콜은 아무런 이유없이 만들어지고 사용되지 않는다.
> - 프로토콜이 존재하는 이유(목적)이 있고, 프로토콜마다 목적과 특징이 각각 다르다.
> - 물론, 패킷의 내용도 달라진다! (IP패킷과 TCP패킷의 내용은 다르다)

- `IP(Internet Protocol)`: 네트워크 간의 주소를 지정한다는 목적을 지닌 프로토콜
- `ARP(Address Resolution Protocol)`: IP주소와 MAC주소를 대응시킨다는 목적을 지닌 프로토콜
- `HTTPS(Hyper Text Transfer Protocol Secure)`: HTTP + 보안
- `TCP(Transmission Control Protocol)`: 신뢰성을 보장하는 전송 프로토콜 (UDP에 비해 신뢰성이 높다)
