# IP주소의 할당: 정적할당과 동적할당(DHCP)

## 정적할당

- 보통 정적 IP주소를 부여하기 위해서는 IP주소, 서브넷 마스크, 게이트웨이(라우터) 주소, DNS 주소 등이 필요하다.
- 하지만 이를 일일이 지정해주는 것은 쉽지 않다.
- 참고로 Gateway는 일반적으로 서로 다른 네트워크를 연결하는 하드웨어/소프트웨어적 수단을 의미한다. 쉽게 `라우터(공유기)` 라고 생각하면 좋다.

## 동적할당 - DHCP(Dynamic Host Configuration Protocol)

**네트워크에 접속하려는 장치에 IP주소를 자동으로 할당해주는 프로토콜**을 의미한다.

### DHCP가 할당해주는 정보

> - DHCP 서버는 호스트에 할당 가능한 IP주소 목록을 관리하다가 IP주소 할당 요청을 받았을 때 IP주소를 할당해주는 호스트이다.
> - 일반적으로 라우터(공유기)가 DHCP 서버의 역할을 수행한다.

- `IP주소`: 네트워크상의 내 주소
- `서브넷 마스크(Subnet Mask)`: 네트워크의 크기 설정
- `기본 게이트웨이(Default Gateway)`: 외부로 나가는 문 (보통은 공유기)
- `DNS 서버 주소`: 도메인 이름을 IP로 바꿔주는 서버의 주소

### 동적 IP주소의 특징

- `사용 가능한 기간(임대 기간)`이 정해져 있다.
- `할당받을 때마다 다른 주소`를 받을 수 있다.

### 동작 방식

- `탐색`: 장치가 네트워크에 "IP주소 줄 수 있는 DHCP 서버 있나요? 라고 브로드캐스트 메시지를 보냄.
- `제안`: 해당하는 DHCP 서버가 특정 주소를 빌려줄 수 있다고 응답.
- `요청`: 빌리겠다고 승낙.
- `확인`: 승낙한 주소로 할당됨

### 임대 시스템

- **주소를 사용할 수 있는 유효 기간이 정해져 있다 (유효기간이 있는 이유는 IP주소를 효율적으로 관리하고 낭비를 막기 위해서!)**
- **임대 시간의 절반 정도가 지나면 장치는 서버에 "이 주소를 계속 써도 될까요?" 라고 물어보고 기간을 연장함.**
- 장치가 네트워크를 나가거나 직접 연결을 끊으면 해당 주소는 다시 서버로 회수되어 다른 장치가 쓸 수 있음.
