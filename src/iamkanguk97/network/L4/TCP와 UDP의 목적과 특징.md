# TCP와 UDP의 목적과 특징

## 포트를 통한 프로세스 식별

> [!NOTE]
>
> - IP주소와 MAC주소는 패킷을 송수신하는 호스트를 특정할 수 있었다.
> - 하지만, 패킷의 최종 송수신 대상은 호스트가 아닌 `호스트가 실행하는 프로세스`이다.
> - ex. **192.168.0.15:8080 에서 콜론 앞부분은 IP주소로 호스트를 식별하고, 콜론 뒷부분은 포트 번호로 애플리케이션 프로세스를 식별한다.**

### 포트 번호

- 16비트로 표현할 수 있는 포트 번호의 총 개수는 2^16 = 65536개이다.
- 포트번호는 크게 잘 알려진 포트, 등록된 포트, 동적 포트로 나뉜다.
- `잘 알려진 포트(Well-Known Port)`: 0~1023번호 (대중적으로 사용되는 애플리케이션을 위한 포트번호 - HTTP, SSH 등)
- `등록된 포트(Registered Port)`: 1024~49151번호 (흔하게 사용되는 애플리케이션을 위한 포트번호 - MySQL, Redis 등)
- `동적 포트(Dynamic Port)`: 49152~65535번호 (자유롭게 사용이 가능한 포트 번호)
  - **서버로서 동작하는 프로그램의 경우 주로 Well-Known Port 혹은 등록된 포트를 사용하는 경우가 많다.**
  - **하지만 클라이언트로서 동작하는 프로그램의 경우에는 동적 포트 번호 중에서 임의의 번호로 할당되는 경우가 많다.**
  - 대표적으로는 `웹 브라우저`가 있다.

## TCP와 UDP의 차이: (비)신뢰성과 (비)연결형 보장

- `TCP`: 신뢰할 수 있는 통신(상태관리/흐름제어/혼잡제어/오류제어) + 연결형 통신(연결 수립 및 종료 과정이 있음)
- `UDP`: 신뢰할 수 없는 통신(제어, 상태관리 없음) + 비연결형 통신(연결 수립 및 종료 과정이 없음)
- **일반적으로 TCP가 UDP보다 신뢰성은 높지만 속도가 떨어짐.**
