# 파일 대신 데이터베이스를 사용하는 이유

## 데이터 일관성 및 무결성 제공이 어렵다.

- 데이터베이스를 한 명의 사용자 혹은 하나의 프로그램만 사용한다는 보장은 없다.
- `Race Condition` 문제가 발생할 여지가 생긴다 (물론 동기화 도구를 사용하면 방지할 수 있지만 번거로움)
- 데이터의 무결성을 보장하기가 어렵다 (ex. 어떤 학생의 나이가 -1이라고 표기가 되어있을 때 파일 시스템에서는 파악이 어렵다)

## 불필요한 중복 저장이 많아진다.

- 파일로 다량의 데이터를 관리할 경우, 불필요한 중복 저장이 발생하기가 쉽다.
- 불필요한 중복 저장은 스노우볼이 되어 큰 저장 공간 낭비로 이어질 수 있다.

## 데이터 변경 시 연관 데이터 변경이 어렵다.

- 특정 파일에서 학과 이름이 변경되었다고 생각해보면, 직접 수동으로 일일이 변경해주어야 한다.

## 정교한 검색이 어렵다.

- 물론 파일에서 데이터 검색이 가능하지만, 보통 파일 내 문자열 검색에 국한되는 경우가 많다.
- "나이가 25살 이상인 컴퓨터과학과인 서울 거주자" 와 같은 정교한 검색은 할 수 없다.

## 백업 및 복구 어려움

- 파일 시스템은 데이터베이스에 비해 부족한 수준으로 지원한다.
