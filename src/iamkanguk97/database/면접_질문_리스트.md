# Q1: 데이터 저장과 관리가 목적이라면 단순히 파일 시스템을 사용해도 될 것입니다. 그럼 데이터 베이스를 사용하는 이유는 무엇일까요?

> [!TIP]
>
> - 파일 시스템은 데이터베이스에 비해 높은 일관성 및 무결성을 제공하기 어렵기 때문이다.
> - 파일 시스템에서는 여러 사용자 또는 프로그램이 동시에 사용하는 경우 Race Condition 문제로 데이터 일관성이 깨지기 쉽다.
> - 그리고 데이터 중복 저장으로 인한 저장 공간의 낭비가 발생하거나, 데이터를 변경할 때 연관된 모든 데이터를 수동으로 변경해야 하는 번거로움이 있다.
> - 그리고 정교한 검색이나 백업 및 복구 기능도 데이터베이스에 비해 미흡하다.
> - 따라서, 대량의 데이터를 관리하며 여러 사용자에게 제공해야 한다면 파일 시스템보다는 데이터베이스를 사용하는 것이 좋다.

# Q2: 트랜잭션과 ACID가 무엇인지 설명해보세요.

> [!TIP]
>
> - 트랜잭션은 Commit이 발생하는 논리적 단위이다.
> - ACID는 안전한 트랜잭션이 지켜야 할 4가지 특성으로 원자성, 일관성, 독립성, 지속성을 의미한다.
> - 원자성(Atomicity): 트랜잭션의 모든 작업이 성공하거나 실패함을 보장하는 성질
> - 일관성(Consistency): 트랜잭션 전후에 데이터베이스의 일관성을 유지하는 성질
> - 격리성(Isolation): 동시에 수행되는 트랜잭션 간의 간섭이 불가능해야 한다.
> - 지속성(Durability): 완료된 트랜잭션의 결과가 영구적으로 반영되는 성질

# Q3: 기본 키가 무엇이며, 어떤 조건을 만족해야 하는지 설명해보세요.

> [!TIP]
>
> `기본키(Primary Key)`는 데이터베이스 테이블에서 각 행을 고유하게 식별하기 위해 선정된 키를 의미한다.
> 유일성을 갖추어 중복되지 않아야 하고, 반드시 값이 존재해야 하기 때문에 NULL 값을 가질 수 없다.

# Q4: 데이터 관리에 있어 때로는 정규화가 필요하지 않을 때도 있다. 정규화가 필요하지 않은 상황은?

> [!TIP]
>
> 정규화는 데이터 일관성과 무결성을 높이지만 테이블을 많이 쪼개다 보면 빈번해지는 Join 연산으로 인해 검색의 성능이 저하될 수 있다.
>
> 따라서, 검색의 성능이 중요한 경우에는 데이터 중복과 번거로움을 감수하더라도 하나의 테이블로 관리하는 역정규화를 고려할 수 있다.
>
> 예를 들어, NoSQL 데이터베이스에서는 성능 최적화를 위해 기본적으로 데이터를 정규화하지 않는다.

# Q5: 인덱스란 무엇이고, 인덱스를 통해 어떻게 성능을 향상시킬 수 있는지 설명해보세요.

> [!TIP]
>
> - 인덱스는 특정 테이블 열에 대한 자료구조로, 검색 속도를 향상시키기 위해 사용된다.
> - 인덱스를 통해 원하는 데이터를 빠르게 조회할 수 있다.
> - 인덱스를 생성하면 해당 테이블의 열 값들이 정렬된 상태로 저장되므로 테이블 전체를 탐색하지 않고도 빠르게 데이터를 찾을 수 있어서 검색 성능이 향상된다.

# Q6: 인덱스가 오히려 성능을 악화시키는 사례도 있습니다. 이러한 성능 악화는 어떤 상황에서 발생하는지, 그래서 인덱스를 언제, 어떻게 사용해야 하는지 설명해보세요.

> [!TIP]
>
> - 인덱스를 사용하면 SELECT 연산의 성능을 향상시킬 수 있지만, INSERT/UPDATE/DELETE 연산의 성능은 저하시킬수도 있다.
> - 이는 해당 연산이 실행될 때 인덱스도 같이 갱신을 해야하기 때문이다.
> - 인덱스는 데이터가 많고 조회가 빈번한 테이블에 사용되고, 중복 데이터가 많지 않은 열이나 JOIN, WHERE, ORDER BY에서 자주 참조되는 열에 생성하는 것이 효율적이다.

# Q7: 인덱스에서는 B-Tree, 혹은 그를 변형한 B+ Tree 같은 자료구조가 사용되는데 이유가 무엇일까요?

> [!TIP]
>
> B 트리 혹은 B+ 트리는 대용량 데이터에 대한 빠른 탐색에 특화된 자료구조이기 때문이다.

# Q8: 참조 무결성 제약 조건에 대해서 설명해보세요.

> [!TIP]
>
> - 외래 키를 통해 테이블 간의 참조 관계에서 데이터의 일관성을 유지하기 위한 조건을 말한다.
> - 외래 키는 참조하는 테이블의 기본 키와 같은 값을 갖거나 NULL을 가져야 한다.

# Q9: INNER JOIN과 OUTER JOIN의 차이점을 설명해보세요.

> [!TIP]
>
> INNER JOIN은 조건을 만족하는 행들만 결과에 포함되고, 공통된 데이터가 있을 경우에만 데이터를 추출한다.
>
> OUTER JOIN은 공통된 값이 없는 행도 포함해서 반환한다.
> 대표적으로는 LEFT OUTER JOIN, RIGHT OUTER JOIN, FULL OUTER JOIN이 있다.
