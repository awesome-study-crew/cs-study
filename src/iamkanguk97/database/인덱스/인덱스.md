# 인덱스

> [!IMPORTANT]
>
> - `인덱스(index)`는 검색 속도 향상을 목적으로 만드는 하나 이상의 테이블 필드에 대한 자료구조
> - 테이블을 처음부터 끝까지 읽는 `Full Table Scan`을 피하고, 조건에 맞는 행의 위치를 빠르게 찾는다.
> - 장점: `WHERE, JOIN, ORDER BY, GROUP BY`의 속도가 빨라질 수 있음
> - 단점: `저장공간 사용` + `INSERT/UPDATE/DELETE` 속도가 느려질 수 있음 (인덱스를 항시 유지해야 하기 때문)

## MySQL InnoDB Index의 자료구조: B+ Tree

MySQL InnoDB의 일반 인덱스는 대부분 B+Tree 계열이다.

### 좋은 이유

- 정렬된 구조
  - 동등 조건 탐색 빠름
  - 범위 검색도 빠름
  - 이미 정렬되어 있는 인덱스를 이용해서 `ORDER BY`도 성능을 뽑아낼 수 있다
