# B-Tree와 B+Tree

### B-Tree(Balanced Tree)란?

- `디스크 기반 시스템(데이터베이스, 파일 시스템 등)`에서 효율적으로 동작하도록 설계된 트리 구조
- `이진 트리`를 확장해서 **하나의 노드가 가질 수 있는 자식 노드의 수가 2개보다 많을 수 있음.**
- B-Tree는 자료를 `정렬된 상태`로 보관한다.
- **왼쪽과 오른쪽 자식 노드의 밸런스를 항상 유지하기 때문에 O(logN)의 시간 복잡도(탐색,삽입,삭제)를 가진다.**

#### 특징

```
        [10 | 20]
       /     |     \
   [5 8]   [12 15]  [25 30]
```

- 하나의 노드에 여러 개의 키를 저장할 수 있음
- 자식 노드는 `키 개수 + 1`개
- 모든 리프는 같은 레벨에 존재
- 정렬 상태 유지

### B+Tree란?

```
         [10 | 20]
        /     |      \
       /      |       \
  ------    -------    -------
 [5 8]   →  [12 15]  →  [25 30]
```

- `B-Tree`의 확장 및 변형구조
