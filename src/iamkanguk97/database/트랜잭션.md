# 트랜잭션과 ACID

## 트랜잭션(Transaction)

- **데이터베이스에서 하나의 논리적 작업 단위**
- 데이터베이스가 처리하는 작업의 단위를 나타내기 때문에 초당 트랜잭션이라는 지표로 데이터베이스의 작업 성능을 나타낸다.

```
(1) A 계좌에서 10만원 차감
(2) B 계좌에 10만원 증가
```

이 두 작업은 반드시 함께 처리가 되어야 하는 부분인데 이렇게 작업을 하나의 단위로 묶는 것을 `트랜잭션`이라고 한다.

## ACID

데이터베이스는 한 명의 사용자나 하나의 프로그램만 데이터베이스를 사용하는 경우가 거의 없다. 여러 사용자 및 프로그램이 동시다발적으로 사용하는 것이 일반적이다.<br/>
그래서 여러 작업을 내포하는 트랜잭션이 동시다발적으로 실행될 때는 안전한 트랜잭션을 보장하기 위해 지켜야 하는 성질이 있다.<br/>
이를 `ACID` 라고 한다.

### Atomicity: 원자성

- 트랜잭션의 모든 작업은 하나의 덩어리처럼 취급된다.
- 하나라도 실패하면 `Rollback`, 모두 성공하면 `Commit`

### Consistency: 일관성

- 트랜잭션 전후에 데이터는 항상 유효한 상태여야 한다.
- 데이터베이스의 무결성 제약조건을 항상 만족해야 한다.
- ex. 계좌 잔액은 0 이상, 기본키는 중복 불가 등

### Isolation: 격리성

- 동시에 실행되는 트랜잭션은 서로 간섭하지 말아야한다.
- **동시에 같은 상품을 구맿거나, 같은 계좌를 수정하는 등 여러 사용자가 동시에 작업할 때 데이터 충돌을 방지해야 한다.**
- DB는 격리 수준(Isolation Level)을 제공한다.
  - Read Uncommitted
  - Read Committed
  - Repeatable Read
  - Serializable
- 격리 수준이 높아질수록 안전하지만 성능은 떨어질 수 있다.

### Durability: 지속성

- 커밋된 데이터는 절대 사라지지 않는다.
- 트랜잭션이 성공적으로 Commit이 된다면 서버가 갑자기 꺼져도 데이터는 보존되어야 한다.
